Quick start

The toolchain needs Node 20 (see .nvmrc; >=20.19). Do not skip nvm use — see the Node note in Triage. 

nvm use                      # Node 20
npm install
cp .env.example .env.local   # then fill in (or: vercel env pull)
# apply supabase/migrations/0001_init.sql through 0008_brand.sql in order in the SQL editor
npm run seed                 # upload placeholder input assets
npm run dev                  # http://localhost:3000 → sign up on /login

Architecture & project structure

Brand Helper follows a modular, feature-first Next.js layout — adapted to our domain and an RSC-first reality (server components + server actions; minimal client state). App code lives under src/; public/, supabase/, scripts/, and examples/ stay at the repo root. 

src/
├─ app/         # routes, layouts, route handlers, error/404 boundaries
├─ features/    # business features (UI + domain logic together)
│  ├─ auth/  brief/  dashboard/  creatives/  runs/
│  ├─ editor/   # Konva in-app brand editor (client-only, ssr:false)
│  ├─ deliverables/ # branded PDF + social-asset builders (react-pdf + sharp)
│  └─ pipeline/ # orchestrator + compositor/compliance/legal/translate/sound
├─ components/  # shared, presentational
│  ├─ ui/       # primitives (Button, Input, Card, Dialog, Tabs…)
│  ├─ layout/   # site-header, footer
│  ├─ feedback/ # toasts, error detail
│  └─ theme/    # theme provider + toggles (the one bit of client state)
├─ services/    # external integrations: supabase/, image/ (AI Gateway)
├─ lib/         # framework-agnostic utils: format, slug, utils, logging
├─ types/       # shared generated types (Supabase database.ts)
└─ config/      # env.ts (server-only) + prohibited-terms
Principle How Brand Helper applies it
App Router Route groups ((app)), nested layouts, route handlers under src/app.
Feature-based Each domain owns its UI + logic in src/features/<domain>; web app and CLI share features/pipeline.
UI vs logic Presentational components in components/; data/logic in features & services; read layer per feature.
Reusable components components/ui (primitives), layout, feedback, theme.
Centralized API layer services/supabase (3-tier clients + storage) and services/image (Vercel AI Gateway — generate + edit).
State management RSC-first — server components + server actions; only theme uses React context. No Redux/Zustand.
Shared utilities lib/ — format, slug, utils, logging.
Strong typing Types inferred from zod schemas; generated DB types in types/database.ts.
Env & config config/env.ts (server-only, lazy) + config/ data; never hardcoded.
Performance Server Components by default; sharp kept external; signed-URL images via next/image.
Code standards Absolute @/ imports, ESLint + Prettier, named exports (no default component exports).

Deliberately omitted (best practice for this app, not cargo-cult): no store/ (no global client state) and no top-level hooks/ — the single hook (useTheme) lives with its provider, and feature hooks live under their feature.

Environment & secrets

Secrets live only in Vercel env vars and a gitignored .env.local; only .env.example (no real values) is committed. Server-only accessors live in src/config/env.ts behind import 'server-only' with lazy getters, so a missing key only errors when a route actually needs it — build and the reuse path work without an AI Gateway key.

Variable Scope Notes
NEXT_PUBLIC_SUPABASE_URL browser Project URL — safe to expose.
NEXT_PUBLIC_SUPABASE_ANON_KEY browser RLS-scoped anon key — safe to expose.
SUPABASE_SERVICE_ROLE_KEY server Bypasses RLS. Never ships to the client. Rotate if ever shared.
AI_GATEWAY_API_KEY server Vercel AI Gateway — image generation + the editor's AI ops. Generate/edit only.
SUPABASE_PROJECT_ID CLI Used only by db:types. Export inline.
PIPELINE_RUN_USER_ID CLI RLS owner UUID for headless pipeline:run.

Triage — symptom → cause → fix

Symptom Cause Fix
npm test crashes at startup: ERR_INVALID_ARG_VALUE … styleText … ['underline','gray'] Wrong Node. Node 21's util.styleText rejects the array form Vitest/rolldown uses. Use Node 20: nvm use. CI pins it via .nvmrc.
tsc errors: Duplicate identifier 'LayoutProps' in .next/types/routes.d 2.ts Stray macOS / cloud-sync " 2" duplicate files inside .next. find .next -name "* 2.*" -delete or rm -rf .next, then re-run.
Dev server shows old colors / radius after switching branches or git stash Turbopack HMR cached a stale globals.css compile. Restart the dev server (full reload won't clear the server cache).
Fontconfig error: Cannot load default config file during tests Benign — sharp font config noise in the test env. Ignore; tests still pass.
npm run db:types errors out SUPABASE_PROJECT_ID not set (intentionally not hardcoded). Run SUPABASE_PROJECT_ID=xxxx npm run db:types. The script writes via a temp file + atomic mv, so a failed run never clobbers the committed src/types/database.ts.
App 500: Missing required environment variable: … .env.local incomplete. Fill it from .env.example (Supabase URL + anon + service role).
Image generation / editor AI op fails (reuse path + canvas edits work) AI_GATEWAY_API_KEY unset — getters are lazy, so only generate/edit error; the editor's AI buttons disable and a 503 is returned. Add AI_GATEWAY_API_KEY, or stick to reuse + non-AI edits for the demo.
Data won't load / schema must be one of… brand_helper schema not exposed in Supabase. Supabase → Project Settings → API → Exposed schemas → add brand_helper.
Locked out at login (rate limited) DB-backed login limiter (hashed-IP key) tripped. Wait out the window, or clear the rate-limit rows in brand_helper.
CI gitleaks job fails A secret-like string was committed. Remove the secret & rotate it; only add genuine false positives to .gitleaks.toml.

Visitor access requests

Brand Helper is invite-only. Prospective visitors request access from a public surface (login, no-access, or landing page); admins review and grant access from the admin area.

  • Request — the public "Request access" modal posts name / organization / email to the submitAccessRequest server action, which rate-limits by IP, validates, and inserts a row into brand_helper.access_requests with the service role (anon has no table grants). If RESEND_API_KEY is set, the admin gets a notification and the requester a confirmation; without it, the request still stores and email is skipped.
  • Review — admins open /admin → Requests tab. Accept sends a magic-link invite (and copies the name/organization onto the member row); Deny closes the request.
  • Manage — the Members tab lists every visitor. Revoke soft-sets members.status = 'revoked' (the app guard bounces them on their next request; admins cannot be revoked); Re-instate restores access.
  • Env — optional Resend vars: RESEND_API_KEY, RESEND_FROM (verified sender), REQUEST_NOTIFY_EMAIL (where new requests are emailed; defaults to the seeded owner). See migration 0010_access_requests.sql.

CI gates

GitHub Actions (.github/workflows/ci.yml) runs on every push and PR. Reproduce locally on Node 20: 

npm run lint
npm run format:check
npm run docs:check     # design-system.md tables in sync with globals.css
npm run typecheck
npm test
npm run build          # needs no live secrets (lazy env)
npm audit --audit-level=high
npm run sbom           # CycloneDX SBOM
# + gitleaks secret scan over full history

Going public — checklist

  • No secrets in the working tree or git history (gitleaks clean over full history).
  • .env.local, .vercel, .next all gitignored — verify with git check-ignore.
  • LICENSE present; README.md reflects public status (not "private").
  • No live backend identifiers hardcoded — db:types reads SUPABASE_PROJECT_ID.
  • npm run docs:check green — docs match the token source of truth.
  • SBOM regenerated; npm audit shows 0 high/critical.
  • Rotate the Supabase service-role key if it was ever shared outside Vercel.
